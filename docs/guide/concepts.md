# 核心概念与架构

理解 Claude Code 的底层原理是高效使用的关键。本章从架构、工作流、上下文管理等维度深入剖析，帮助你从"会用"进阶到"用好"。

## 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户 (User)                               │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                    CLI 交互层 (Node.js)                          │
│  - 命令解析 (/commands, --model, --agent)                       │
│  - 会话管理 (历史、压缩、恢复)                                    │
│  - 输出渲染 (Markdown、diff、进度条)                              │
│  - 权限系统 (自动确认、危险命令拦截)                               │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                    Agent 引擎                                    │
│  - 任务规划 (分解为子任务，使用 TodoWrite 跟踪)                   │
│  - 工具调用 (Read、Write、Bash、Grep 等)                         │
│  - 上下文管理 (自动压缩、CLAUDE.md 加载)                          │
│  - Agent 调度 (自定义 Agent、builtin Agent)                      │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                    Claude API (Anthropic)                        │
│  - 消息处理 (user/assistant 消息交替)                             │
│  - 工具定义 (functions calling)                                  │
│  - 响应生成 (流式输出)                                            │
│  - 上下文窗口 (100 万 token)                                      │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                    本地环境 (沙箱)                                │
│  - 文件系统 (Read/Write/Edit)                                    │
│  - Git 操作 (status、commit、push)                               │
│  - Shell 命令 (Bash)                                             │
│  - MCP 工具 (外部服务集成)                                        │
└─────────────────────────────────────────────────────────────────┘
```

## 工作模式对比

Claude Code 支持多种启动和使用方式，适用于不同场景：

| 模式 | 命令 | 特点 | 适用场景 |
|------|------|------|----------|
| **交互模式** | `claude` | 持续对话，保留上下文 | 复杂任务、多轮迭代 |
| **单次执行** | `claude -p "prompt"` | 执行后退出，无状态 | 脚本集成、CI/CD |
| **会话继续** | `claude -c` 或 `claude --continue` | 继续最近一次会话 | 快速回到之前的工作 |
| **会话恢复** | `claude -r [session]` | 恢复指定会话 | 继续未完成的任务 |
| **PR 关联** | `claude --from-pr [num]` | 从 PR 恢复会话 | Code Review 相关工作 |

**交互模式示例**：

```bash
cd /path/to/project
claude
# 启动后进入持续对话
```

**单次执行示例**：

```bash
# 解释代码
claude -p "解释 src/index.ts 中的 main 函数"

# 代码审查并输出到文件
claude -p "审查 src/auth/login.ts" > review.txt

# 带预算限制
claude -p --max-budget-usd 0.05 "生成一个 useLocalStorage React Hook"
```

**会话恢复示例**：

```bash
# 继续最近一次会话
claude -c

# 通过名称恢复（交互选择）
claude -r

# 恢复指定会话
claude -r auth-feature
```

## 上下文管理

### 上下文组成

Claude Code 的上下文窗口为 **100 万 token**（1M），组成如下：

```
┌──────────────────────────────────────────────┐
│  系统提示词 (System Prompt)           ~5K     │
├──────────────────────────────────────────────┤
│  CLAUDE.md (项目配置)                ~2K    │
├──────────────────────────────────────────────┤
│  Memory (持久化记忆)                ~5-10K  │
├──────────────────────────────────────────────┤
│  工具定义 (Tools/MCP)               ~20-50K │
├──────────────────────────────────────────────┤
│  会话历史 (对话+工具结果)           ~918K   │
└──────────────────────────────────────────────┘
```

### 自动压缩机制

当上下文接近上限时，Claude Code 会：

1. **识别关键信息**：保留 CLAUDE.md、Rules、最近的对话
2. **压缩旧内容**：将早期对话提炼为摘要
3. **丢弃冗余**：移除重复的工具调用结果

**手动压缩操作**：

| 操作 | 命令 | 效果 | 适用场景 |
|------|------|------|----------|
| `/clear` | 手动 | 清空所有历史，完全重置 | 开始全新任务 |
| `/compact` | 手动 | 保留关键上下文，压缩细节 | 任务进行到里程碑 |
| 自动压缩 | 阈值触发 | 同上，但不可控 | 防止超限 |

**最佳实践**：在逻辑断点手动执行 `/compact`，而非依赖自动压缩。

### 上下文优化配置

通过 `--effort` 选项控制任务投入程度：

```bash
# 低投入：快速简单任务
claude --effort low

# 中等投入：日常开发（默认）
claude --effort medium

# 高投入：复杂重构
claude --effort high

# 最大投入：架构设计、复杂问题
claude --effort max
```

## 工具调用机制

Claude Code 通过工具与外部世界交互。每个工具调用是一个标准化的 JSON 请求。

### 工具调用流程

```
1. 用户发送请求
         │
         ▼
2. Claude 分析 → 决策需要调用工具
         │
         ▼
3. 返回 tool_use 消息
   {
     "type": "tool_use",
     "name": "Read",
     "input": { "file_path": "/src/index.ts" }
   }
         │
         ▼
4. 系统执行工具（沙箱环境）
         │
         ▼
5. 返回 tool_result 消息
   {
     "type": "tool_result",
     "tool_use_id": "...",
     "content": "文件内容..."
   }
         │
         ▼
6. Claude 分析结果 → 继续或最终响应
```

### 内置工具详解

| 工具 | 功能 | 安全级别 |
|------|------|----------|
| **Read** | 读取文件 | 只读，无风险 |
| **Write** | 创建/覆盖文件 | 需要确认 |
| **Edit** | 精确替换字符串 | 需要确认 |
| **Bash** | 执行 Shell 命令 | 按命令类型分级 |
| **Glob** | 模式匹配查找文件 | 只读，无风险 |
| **Grep** | 内容搜索 | 只读，无风险 |
| **TodoWrite** | 管理任务列表 | 无风险 |
| **AskUserQuestion** | 向用户提问 | 无风险 |
| **WebSearch** | 网络搜索 | 无风险 |
| **WebFetch** | 抓取网页内容 | 无风险 |
| **NotebookEdit** | 编辑 Jupyter notebook | 需要确认 |

**工具签名**（通过 `claude --tools` 可限制可用工具）：

```
Read(file_path, limit?, offset?)
Write(file_path, content)
Edit(file_path, old_string, new_string)
Bash(command, timeout?, description?)
Glob(pattern, path?)
Grep(pattern, path?, output_mode?)
TodoWrite(todos)
AskUserQuestion(questions)
WebSearch(query)
WebFetch(url, prompt)
```

### 并行工具调用

Claude Code 支持**在同一轮响应中调用多个独立工具**，大幅提升效率：

```json
[
  {"type": "tool_use", "name": "Read", "input": {"file_path": "/src/a.ts"}},
  {"type": "tool_use", "name": "Read", "input": {"file_path": "/src/b.ts"}},
  {"type": "tool_use", "name": "Read", "input": {"file_path": "/src/c.ts"}}
]
```

## Agent 引擎详解

### Agent 决策循环

```
┌─────────────────────────────────────────────────────────────┐
│                    收到用户任务                              │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  阶段 1: 分析 (Analyze)                                      │
│  - 理解用户意图                                              │
│  - 识别所需工具                                              │
│  - 评估任务复杂度                                            │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  阶段 2: 规划 (Plan)                                         │
│  - 分解为子任务                                              │
│  - 确定执行顺序                                              │
│  - 使用 TodoWrite 跟踪进度                                  │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  阶段 3: 执行 (Execute)                                      │
│  - 调用工具                                                  │
│  - 收集结果                                                  │
│  - 处理错误                                                  │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  阶段 4: 验证 (Verify)                                       │
│  - 检查结果是否符合预期                                       │
│  - 运行测试/类型检查                                          │
│  - 必要时回退或调整                                          │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
                    是否还有未完成的子任务？
                    ├── 是 → 回到阶段 3
                    └── 否 → 输出最终结果
```

### Agent 配置方式

Claude Code 的 Agent 通过 JSON 定义，不使用 YAML 文件：

```json
{
  "agents": {
    "review": {
      "description": "专业的代码审查专家",
      "prompt": "你是代码审查专家，专注以下方面：\n1. 类型安全\n2. 错误处理\n3. 性能问题\n4. 安全漏洞"
    }
  }
}
```

通过 `--agents` 选项在启动时传入 JSON，或将配置写入 `~/.claude/settings.json` 持久化。

### 内置 Agent

Claude Code 提供了 5 个内置 Agent：

- **claude-code-guide** (haiku) — 专门学习 Claude Code 使用
- **Explore** (haiku) — 快速探索代码库
- **general-purpose** (inherit) — 通用任务
- **Plan** (inherit) — 复杂任务规划
- **statusline-setup** (sonnet) — 状态栏配置

## MCP 协议集成

MCP (Model Context Protocol) 是连接 AI 与外部工具的标准协议。

### 架构对比

**无 MCP**：
```
Claude Code → 内置工具 (Read, Write, Bash) → 本地环境
```

**有 MCP**：
```
Claude Code → MCP Client → MCP Server → 外部服务
                              ├── GitHub API
                              ├── PostgreSQL
                              ├── Slack
                              └── 任意自定义服务
```

### MCP 服务器管理

MCP 服务器通过 `claude mcp` 命令管理：

```bash
claude mcp list              # 列出已配置的服务器
claude mcp add <name> ...    # 添加服务器
claude mcp remove <name>     # 移除服务器
claude mcp add-from-claude-desktop  # 从 Claude Desktop 导入
```

## Rules 规则引擎

Rules 是始终生效的指令，告诉 Claude 如何行为。详见 [CLAUDE.md 规则系统](/guide/rules)。

### 规则加载顺序

```
项目级 Rules (.claude/rules/*.md)     ← 最高优先级
         │
         ▼
用户级 Rules (~/.claude/rules/*.md)
         │
         ▼
CLAUDE.md 中的规则
         │
         ▼
默认行为                           ← 最低优先级
```

### 规则示例（符合真实格式）

```markdown
# .claude/rules/testing.md

- 每个测试文件必须对应一个源文件
- 使用 describe/it 结构
- 测试命名: "should <预期行为> when <条件>"
- 覆盖率目标 > 80%
```

## Hooks 钩子系统

Hooks 在特定事件发生时**自动执行脚本**，用于工作流自动化。

### 支持的事件

| 事件 | 触发时机 | 典型用途 |
|------|----------|----------|
| `Attach` | Agent 连接时 | 加载上下文、权限检查 |
| `Detach` | Agent 断开时 | 清理资源、生成报告 |
| `User` | 用户消息发送前 | 内容审核、输入转换 |
| `Bot` | Claude 响应前 | 过滤输出、添加注释 |
| `Stop` | Agent 停止时 | 会话总结、状态保存 |
| `Permissions` | 权限检查时 | 自定义权限策略 |
| `MobileResume` | 移动端恢复时 | 上下文同步 |
| `McpCallback` | MCP 回调时 | MCP 审计 |

### Hook 配置（settings.json）

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "true",
        "hooks": [
          {
            "type": "command",
            "command": "echo '会话结束，执行清理任务'"
          }
        ]
      }
    ]
  }
}
```

## Memory 记忆系统

### 记忆类型

| 类型 | 存储位置 | 生命周期 | 用途 |
|------|----------|----------|------|
| 会话记忆 | 内存 | 当前会话 | 临时上下文 |
| 项目记忆 | `~/.claude/projects/` | 项目生命周期 | 项目特定信息 |
| 用户记忆 | `~/.claude/memory/` | 永久 | 个人偏好 |

### 手动记忆

Claude Code 支持通过对话记住信息：

```
记住：此项目使用 pnpm 作为包管理器
记住：用户偏好中文回复
```

### 自动记忆（Auto-Memory）

Claude Code 会自动记住：
- 项目结构和技术栈
- 代码风格规范
- 最近的修改历史
- 解决过的问题和方案

通过 `~/.claude/settings.json` 中的配置可以控制自动记忆行为。

## 性能与成本

### Token 消耗估算

| 操作 | 平均 Token | 成本 (Sonnet) | 成本 (Opus) |
|------|-----------|--------------|-------------|
| 读取 100 行代码 | ~500 | $0.001 | $0.003 |
| 编辑 50 行代码 | ~1,000 | $0.002 | $0.006 |
| 运行一次测试 | ~2,000 | $0.004 | $0.012 |
| 复杂重构任务 | 10,000+ | $0.02 | $0.06 |

### 优化策略

1. **默认使用 Sonnet**：覆盖 80% 任务，成本降低 60%
2. **限制任务投入**：`--effort medium`（默认）
3. **主动压缩**：里程碑处手动 `/compact`
4. **禁用无用 MCP**：每个 MCP 服务器的工具定义消耗 token
5. **使用预算限制**：`--max-budget-usd 0.05`

## 总结

| 概念 | 一句话解释 | 关键作用 |
|------|-----------|----------|
| Agent | 经过专门配置的 Claude 实例 | 实现专业化工作流 |
| Tool | 与外部环境交互的能力 | 读文件、执行命令 |
| MCP | 连接外部服务的标准协议 | 扩展能力边界 |
| Rule | 始终生效的行为指令 | 确保一致性 |
| Hook | 事件触发的自动化脚本 | 工作流集成 |
| Memory | 跨会话持久化信息 | 积累上下文 |
| Context | 100 万 token 的对话空间 | 维持任务连续性 |
