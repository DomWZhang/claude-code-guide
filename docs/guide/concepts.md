# 核心概念

深入理解 Claude Code 的核心概念，包括工作原理和架构设计。

## Claude Code 架构

Claude Code 是一个基于终端的 AI 编程助手，其核心架构如下：

```
┌─────────────────────────────────────────────────────────┐
│                    用户 (You)                           │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│                  CLI 交互层 (Node.js)                    │
│  - 命令行解析                                             │
│  - 会话管理                                              │
│  - 输出渲染                                              │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│                   Agent 引擎                             │
│  - 任务规划                                              │
│  - 工具调用                                              │
│  - 上下文管理                                            │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│                   Claude API                            │
│  - 消息处理                                              │
│  - 工具定义                                              │
│  - 响应生成                                              │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│                   本地环境                               │
│  - 文件系统                                              │
│  - Git                                                   │
│  - Shell                                                 │
│  - MCP 工具                                              │
└─────────────────────────────────────────────────────────┘
```

## 工作模式

### 单轮交互

用户发起请求，Claude 处理并返回结果：

```
用户 → 请求 → Claude → 结果
```

### 多轮对话

持续交互，积累上下文：

```
用户 → 请求1 → Claude → 结果1
  ↓
用户 → 请求2 → Claude → 结果2
  ↓
用户 → 请求3 → Claude → 结果3
```

### Agent 模式

Claude 自主决策并执行多步操作：

```
用户 → 复杂任务
  ↓
Claude → 分析任务
  ↓
Claude → 规划步骤
  ↓
Claude → 执行步骤1 → 验证 → 反馈
  ↓
Claude → 执行步骤2 → 验证 → 反馈
  ↓
...
  ↓
Claude → 任务完成
```

## 上下文管理

### 上下文窗口

Claude Code 支持高达 **100 万 token** 的上下文窗口：

- 可以一次性加载整个代码库
- 支持长时间对话不丢失上下文
- 自动上下文压缩（当接近上限时）

### 上下文组成

```

├────────────────────────────────────────┤
│            系统提示词                   │
├────────────────────────────────────────┤
│            CLAUDE.md                   │
├────────────────────────────────────────┤
│            项目文件内容                 │
├────────────────────────────────────────┤
│            Memory 记忆                 │
├────────────────────────────────────────┤
│            MCP 工具定义                 │

```

## 工具调用

Claude Code 的核心能力之一是通过工具与外部世界交互。

### 工具调用流程

```
1. 用户发送请求
         ↓
2. Claude 分析请求，决定是否需要调用工具
         ↓
3. Claude 发送 tool_use 请求
         ↓
4. 系统执行工具
         ↓
5. 工具返回结果
         ↓
6. Claude 收到结果，继续处理
         ↓
7. 最终返回自然语言响应
```

### 工具调用示例

```json
{
  "type": "tool_use",
  "name": "Read",
  "input": {
    "file_path": "/src/app.ts"
  }
}
```

响应：

```json
{
  "type": "tool_result",
  "tool_use_id": "tool_123",
  "content": "文件内容..."
}
```

## 项目感知

Claude Code 能够理解项目结构和上下文。

### 项目类型识别

自动识别项目类型：

- **JavaScript/TypeScript** - 检测 package.json、tsconfig.json
- **Python** - 检测 requirements.txt、pyproject.toml
- **Java** - 检测 pom.xml、build.gradle
- **Go** - 检测 go.mod
- **Rust** - 检测 Cargo.toml

### 依赖分析

理解模块间的依赖关系：

```
src/
├── components/    ← 可独立理解
│   ├── Button.tsx
│   └── Modal.tsx
├── hooks/         ← 依赖 components
│   └── useModal.ts
└── pages/         ← 依赖 hooks, components
    └── Home.tsx
```

## 安全机制

### 沙箱执行

- 终端命令在受控环境中执行
- 危险操作需要用户确认
- 可配置的权限级别

### 确认机制

```
⚠️ 警告：以下操作不可撤销
- 删除文件
- 覆盖现有代码
- 执行破坏性 Git 操作
```

### 权限控制

通过 hooks 实现细粒度权限控制。

## 会话隔离

每个会话独立：

- 不同的上下文历史
- 独立的记忆系统
- 独立的配置覆盖

## 错误处理

Claude Code 具有完善的错误处理机制：

1. **语法错误** - 指出并建议修复
2. **类型错误** - 提供类型修正建议
3. **运行时错误** - 分析错误原因
4. **逻辑错误** - 重构建议

## 性能优化

### 上下文优化

- 只加载相关文件
- 自动压缩冗余内容
- 智能文件选择

### 工具调用优化

- 批量操作
- 缓存常用结果
- 并行工具调用
