# 基础使用

本节介绍 Claude Code 的日常使用技巧和常用操作。

## 交互模式

Claude Code 支持两种交互模式：

### 交互模式

启动后持续对话：

```bash
claude
```

在交互模式下，你可以：
- 随时提出问题
- 逐步指导 Claude 完成复杂任务
- 使用 Slash 命令
- 实时查看执行结果

### 单次执行模式

执行单个任务后退出：

```bash
claude -p "解释 src/app.ts 中的业务逻辑"
```

使用 `--print` 或 `-p` 参数：

```bash
claude -p "修复 src/index.ts 中的 bug"
```

## 发起请求

### 描述性请求

越详细的描述，Claude Code 越能准确理解你的需求：

```bash
# ❌ 模糊请求
"修复 bug"

# ✅ 清晰请求
"修复 src/api/user.ts 中 getUserById 函数的类型错误，
当传入无效 ID 时应该返回 null 而不是抛出异常"

# ✅ 非常详细的请求
"在 src/components/ 目录下创建一个 UserCard 组件，
使用 TypeScript，包含头像、用户名、邮箱显示，
支持 hover 效果，遵循项目的组件规范"
```

### 指定范围

明确指定文件或目录：

```
读取 /src/utils/helpers.ts
```

```
修改 src/components/ 下所有组件的类型定义
```

```
分析 src/services/api.ts 的实现
```

## 任务执行流程

当你发起请求时，Claude Code 会：

1. **理解意图** - 分析你的需求
2. **分析环境** - 读取相关文件、理解项目结构
3. **制定计划** - 规划操作步骤
4. **执行操作** - 使用各种工具完成任务
5. **确认结果** - 汇报完成情况

## 查看 Claude 的思考

Claude Code 在执行复杂任务时会显示其思考过程：

```
我需要完成这个任务：
1. 先读取相关文件了解当前状态
2. 分析需要修改的部分
3. 进行修改
4. 验证修改结果
```

这能帮助你：
- 理解 Claude 的决策过程
- 及时纠正误解
- 更好地指导 Claude


## 会话管理

### 新建会话

```
/new
```

### 查看历史

Claude Code 会自动保存会话历史。

### 上下文窗口

Claude Code 使用 1M token 的上下文窗口，可以处理大量文件。

当上下文接近上限时，Claude 会自动压缩旧内容。

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + C` | 取消当前操作 |
| `Ctrl + D` | 退出 Claude Code |
| `Ctrl + L` | 清屏 |
| `↑ / ↓` | 浏览历史命令 |

::: tip 高效使用建议
1. **保持上下文清晰** - 每次新任务时，说明项目背景
2. **分步骤进行** - 复杂任务分解为多个简单步骤
3. **及时反馈** - 错误时立即指出，帮助 Claude 纠正
4. **善用指定范围** - 告诉 Claude 具体要操作的文件
5. **保持耐心** - 复杂任务需要多轮交互
:::
